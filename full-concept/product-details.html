About this build

Sentry API is a production-ready authentication and authorisation starter for Spring Boot 3.2+. Drop it into any new or existing project and ship JWT auth, OAuth2 (Google, GitHub, Microsoft), role-based access control, audit logging and rate-limiting in under an hour.

Built by SpringForge — the team behind 14 of Forge's top-rated backend builds, now used in production by 3,800+ teams including FinTech, HealthTech and govtech orgs across 6 continents.

What's inside the .zip

  • Spring Boot 3.2 + Java 21 monorepo with Gradle Kotlin DSL
  • JWT issuer + refresh token rotation (RS256, JWKs endpoint)
  • OAuth2 social login: Google, GitHub, Microsoft, Apple
  • Role-based access control with hierarchical permissions
  • Argon2id password hashing (zero-config defaults)
  • Audit log table + Kafka emitter for SIEM streaming
  • Rate limiting (Bucket4j) with Redis-backed sliding window
  • Docker Compose for Postgres, Redis, Mailpit
  • Postman collection & OpenAPI 3 spec
  • 92% line coverage · JUnit 5 + Testcontainers
  • GitHub Actions CI workflow included
  • Complete deployment guides (Render, Fly.io, AWS ECS)

Feature highlights

Every feature ships fully wired into the demo app — clone, run, and click around. No "exercise left to the reader" code.

🔐 Authentication

  • Email + password sign-up with email verification flow
  • Argon2id password hashing with configurable cost
  • JWT access tokens (RS256) + opaque refresh tokens
  • WebAuthn / passkeys support out of the box
  • Magic link sign-in (Mailpit dev, SendGrid prod)
  • OAuth2 social: Google, GitHub, Microsoft, Apple, LinkedIn

👤 Authorization & RBAC

  • Hierarchical role & permission system
  • Method-level @PreAuthorize annotations with custom SpEL
  • Per-tenant scoping for multi-tenant SaaS use
  • Row-level security helpers for Postgres

🛡 Security & compliance

  • OWASP-aligned defaults (CSRF, CORS, headers, rate limits)
  • 2FA via TOTP (Google Authenticator, 1Password)
  • Brute-force protection with Redis sliding window
  • Full audit trail of every auth event
  • SOC 2 evidence-friendly logging format

📊 Operations

  • Spring Actuator endpoints (health, metrics, prometheus)
  • Structured JSON logs (Logback + Logstash encoder)
  • Open-telemetry instrumentation included
  • One-line Sentry.io error tracking

Setup & deploy

From git clone to running locally: about 4 minutes. From local to a deployed staging environment: about 14.

① Local development

git clone https://github.com/your-org/sentry-api.git
cd sentry-api
cp .env.example .env
docker compose up -d        # postgres, redis, mailpit
./gradlew bootRun           # app on :8080, docs on :8080/swagger

② Run the test suite

./gradlew test
# BUILD SUCCESSFUL · 142 tests · 92% coverage

③ Deploy to production

Pick your platform — guides included for each:

  • Render — Dockerfile + render.yaml ready, one-click deploy
  • Fly.io — fly.toml + Dockerfile, 4-region rollout walkthrough
  • AWS ECS — Terraform module, ALB + RDS + ElastiCache
  • Kubernetes — Helm chart with HPA & PodDisruptionBudget

④ Connect a frontend

Drop-in clients shipped for: React (React Query + Zustand), Vue 3 (Pinia), Svelte (Stores), and a Postman environment for QA.
